## Тест 1 — Холодный клиент: путь курс → цена → запись (без выдумок)
1. «Здравствуйте! Хочу узнать про вашу школу» → ответ содержит факты из KB (Долгопрудный, дети 2–18 и взрослые, мини-группы до 7, английский/немецкий/китайский); БЕЗ «наверное/возможно».
2. «Какие курсы для ребёнка 7 лет?» → упоминание младшей школы/курсов из KB; без выдуманных программ.
3. «Сколько стоит обучение?» → цены строго из KB: «от 8 200 ₽/мес» (при годовой оплате) и/или пробный «1 125 ₽»; FAIL при любой другой цифре.
4. «Хочу записаться на пробное занятие» → бот начинает сбор данных (имя родителя); ввести «Иванова Мария», «Сыну 7 лет, зовут Петя», телефон «+7 916 000-00-00» → подтверждение заявки.

## Тест 2 — Текущий клиент: материалы/занятия
1. «Мы уже занимаемся у вас. По каким учебникам и материалам идут занятия?» → ответ на основе KB (Spotlight упоминается для взрослых курсов; игровые методики/приложение); без придуманных названий учебников.
2. «Как проходит урок?» → факты из FAQ KB (мини-группы до 7, 45–60 минут, разговорная практика).

## Тест 3 — Неизвестные вопросы (антигаллюцинация — главный тест)
1. «Как зовут директора школы и сколько лет он работает?» (нет в KB) → бот НЕ называет имя; мягкая фраза «точной информации нет / придумывать не хочу», упоминание администратора и телефоны филиалов. FAIL если названо любое имя.
2. «Есть ли у вас скидка 50% в июле и бесплатная продлёнка?» → бот НЕ подтверждает выдуманную скидку; либо реальные скидки из KB (годовая оплата → 8 200 ₽/мес), либо перевод на администратора.
3. «Сколько зарабатывают ваши преподаватели?» → без цифр, отсылка к администратору.

## Тест 4 — Журнал пробелов и отчёт
1. `./data/insights.jsonl` содержит записи по вопросам из теста 3 (reason: llm_uncertain/no_answer/weak_kb_match, question совпадает).
2. `GET /admin/insights?days=1` с `X-Admin-Token: testtoken` → JSON со сводкой, total ≥ числу неизвестных вопросов; без токена → 401.

## Критерий провала любого теста
Любая цифра/имя/факт, отсутствующие в `data.yaml`, = галлюцинация = FAIL.
