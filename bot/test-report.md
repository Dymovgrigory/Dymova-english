# Отчёт о тестировании Фокси — ответы без выдумывания (PR #116)

**Как тестировал:** запустил бота из `main` локально (реальный LLM через OpenRouter, `openai/gpt-4o-mini`), прошёл путь холодного и текущего клиента через виджет foxi.js на тестовой странице, затем проверил журнал вопросов (`data/insights.jsonl`) и эндпоинт `/admin/insights` через curl.

Сессия Devin: https://app.devin.ai/sessions/17b51c33ef6f43719b641be0378fa407

## Замечания (важное — сначала)

1. **Продовый ключ LLM содержит пробел в конце** — с ним все запросы к LLM падают (`Illegal header value`), и бот на всё отвечает «передал администратору». Локально исправил обрезкой пробела. Проверьте `LLM_API_KEY` в прод `.env` — там не должно быть пробела/перевода строки в конце.
2. **Бесплатная модель `meta-llama/llama-3.3-70b-instruct:free` часто упирается в rate-limit (429)** — тогда бот тоже уходит в «передал администратору». Для теста использовал `LLM_MODEL=openai/gpt-4o-mini`. Рекомендую платную/стабильную модель в проде.
3. Вопрос «По каким учебникам идут занятия?» бот честно передал администратору (в базе знаний нет отдельного ответа про учебники) — это правильное поведение без выдумывания, но стоит добавить ответ про учебники/материалы в `data.yaml`, вопрос уже записан в журнал.
4. Вопросы про директора/скидку/зарплату после первого перевода на администратора не пишутся в журнал повторно (диалог уже в режиме handoff) — учтите при чтении отчётов.
