## Технические заметки

### Как вставить кастомный блок в Tilda
1. В редакторе страницы нажать «+» (добавить блок)
2. Выбрать «Другое» -> «HTML-код» (блок T123)
3. Вставить содержимое файла `tilda_*.html` целиком (HTML + CSS + JS)
4. Сохранить и опубликовать

### Как подключить форму заявки через нативный попап Tilda
1. Добавить блок формы с popup (категория «Форма» в библиотеке)
2. В настройках формы убедиться, что используются настройки проекта (не кастомные)
3. Добавить скрытое поле `source` со значением, указывающим блок/кнопку

#### Технические заметки (важно для будущих сессий):
- **Редактор Tilda открывается только с параметром `&edit=y`**: `https://tilda.ru/page/?pageid=151210576&projectid=2053071&edit=y`. Без него страница рендерится пустой.
- **API сортировки блоков**: `comm=saverecordssort` принимает порядок как **массив** `sorts[0]=<id>&sorts[1]=<id>&...`, а НЕ как CSV `sort=`. Параметр `sort=` молча игнорируется (сервер всё равно отвечает `OK`). Команда найдена перехватом запроса штатной стрелки перемещения блока в редакторе.
- После вызова `saverecordssort` нужно **перезагрузить редактор** (F5), чтобы он подхватил серверный порядок, затем публиковать — иначе кнопка «Опубликовать» зальёт старый in-memory порядок.
- Скрытие/показ блоков на странице: `.r.t-rec{display:none!important}` + `.r.t-rec:has([id^="fxb-"]){display:block!important}` — любой блок с id, начинающимся на `fxb-`, показывается автоматически (поэтому новый `fxb-reviews` отобразился без доп. настроек).
- `comm=deleterecord` (pageid+recordid) — рабочее удаление блока.
- Скрипты: `tilda_reorder.py` (переупорядочивание в зафиксированный порядок), `tilda_upload_blocks.py` (список блоков обновлён под новый стандарт).
